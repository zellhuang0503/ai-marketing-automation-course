# 梵亞行銷：10 天 AI 自動化實戰特訓營 (AntiGravity + 伯大尼實務)
**副標題：告別 n8n，用 10 天打造「屏東伯大尼」的 24h 全自主 AI 行銷團隊**

本課程專為實戰派行銷人設計，**70% 以上為實作環節**。以「屏東伯大尼靈修院」真實的社群營運、捐款籌募、活動推廣需求為核心場景。學員將在 10 天內，親手利用 AntiGravity 的「Skill (技能)」與「Workflow (工作流)」，打造出能自動化產出圖文、監測資訊、回覆訊息的多智能體行銷系統。

---

## 🎯 課程特色
* **真實場景導向：** 完全捨棄空泛理論，直接套用「屏東伯大尼」的營運痛點做為實戰演練。
* **高產現實作：** 超過 7 成時間，學員都在動手寫 `.agents/workflows`、建 Skills，並看著 AI 自動產出成果。
* **無痛程式過渡：** 用 AI 聽得懂的自然語言（Workflow Scripting）取代 n8n 複雜的節點與 API 參數。

---

## 📅 課程大綱 (共 10 天實戰，每日建議 1-2 小時)

### 【階段一：基地建設與角色指派 (環境與心法)】

**Day 1: 告別節點圖，從 n8n 轉向 AI 代理人網路**
* **理論 (30%)：**
    * 解析為什麼傳統自動化 (n8n/Make) 已死？
    * 認識 AntiGravity 的多智能體架構 (Jarvis 核心系統)。
    * 了解 Workspace 目錄結構：`/docs` (記憶/研究), `/src` (產出), `.agents` (配置)。
* **實作 (70%) - 伯大尼環境初始化：**
    * 安裝與啟動 AntiGravity IDE。
    * 建立「屏東伯大尼行銷專案」並配置基本工作目錄。
    * 撰寫第一份 `Jarvis` 核心系統提示詞 (設定為伯大尼行銷總監)。

**Day 2: 喚醒你的團隊：Marcus, Oliver, Jimmy, Jason, Patric**
* **理論 (30%)：**
    * 多智能體協作協議 (Vanya AI Agency Protocol) 運作原理。
* **實作 (70%) - 伯大尼團隊配置：**
    * 撰寫 Jimmy (文案) 的 Prompt 框架：限定「繁體中文」、「溫馨、感人、公益呼籲」語氣。
    * 撰寫 Marcus (市調) 的檢索邏輯，用來抓取其他公益團體的活動資訊。
    * 測試：請 Jarvis 指派 Jimmy 為伯大尼的「新建工程募款」寫第一篇測試文案。

### 【階段二：鍛造行銷武器庫 (Skills 實作)】

**Day 3: 社群自動轉化術 (長文轉 IG 圖文/FB 貼文) Skill**
* **實作場景：** 伯大尼院長給了一篇 1500 字的年度感恩回顧，需要立刻產出 FB 貼文與 IG 滑動圖文腳本。
* **實作重點：**
    * 在 `.agents/skills/content_adapter/SKILL.md` 建立技能。
    * 定義轉換邏輯與字數限制。
    * **實測：** 丟入一篇長文，指令觸發，看 Agent 自動重寫並分段。

**Day 4: 活動情報自動爬蟲 (競品與熱門時事監測) Skill**
* **實作場景：** 隨時監測衛福部最新政策、或是屏東其他社福機構的活動，作為搭蹭話題的依據。
* **實作重點：**
    * 教導 Agent 使用 `browser_action` 或 `tavily_search`。
    * 建立 `.agents/skills/event_monitor/SKILL.md`。
    * **實測：** 下達指令：「Marcus，幫我查這週內台灣社福界最熱門的募款話題，並總結 3 個我們可以參考的點子。」

**Day 5: 捐款感謝函與問卷自動客製化 Skill**
* **實作場景：** 收到不同金額與頻率的捐款名單，需要自動生成有溫度的個人化感謝信 (EDM)。
* **實作重點：**
    * 在 `./docs` 置入一份模擬的 Excel/CSV 捐款者輪廓名單。
    * 建立 `donor_gratitude` Skill，設定依據「捐款金額/歷史」生成特定語氣。
    * **實測：** 讓 Jimmy 讀取名單，一口氣產出 10 封針對不同對象的專屬 Email。

### 【階段三：打通任督二脈 (Workflows 實作)】

**Day 6: 第一個行銷自動化流程：晨間社群快報**
* **實作場景：** 每天早上 9 點，自動搜尋當日適合發佈的伯大尼相關時事，並準備好草稿。
* **實作重點：**
    * 在 `.agents/workflows/morning_briefing.md` 建立流程。
    * 使用 `// turbo` 標記，實現免審批全自動執行。
    * **實測：** 觸發 Workflow，觀察 Jarvis 如何指揮 Marcus 搜尋，再轉交給 Jimmy 寫草稿。

**Day 7: 一條龍產出：從主題企劃到多平台文案 (Content Pipeline)**
* **實作場景：** 伯大尼即將舉辦「中秋節家屬聯誼」，需要一份完整企劃、FB 貼文、LINE @ 推播、新聞稿。
* **實作重點：**
    * 編寫 `campaign_pipeline.md` 工作流。
    * 定義步驟：1. 讀取背景資料 -> 2. 構思主題 -> 3. 產出 FB -> 4. 產出 LINE -> 5. 產出新聞稿。
    * **實測：** 只要丟入一句「我們要辦中秋晚會，主題是團圓」，五分鐘後驗收全套行銷包。

**Day 8: 圖文並茂：設計企劃與 UI 原型預覽 (Oliver 的逆襲)**
* **實作場景：** 為「新建大樓募資專案」規劃一個 Landing Page (募款一頁式網站)。
* **實作重點：**
    * 使喚 Oliver (設計) 與 Jason (程式)。
    * 讓 Agent 讀取募款文案，自動生成包含 Shadcn UI / TailwindCSS 的前端代碼。
    * **實測：** 執行 `npm run dev` (交由 Jason 啟動)，在本機 `/dist` 或 local server 預覽募款網站的視覺雛形，若不符合「刻意極簡」風格，立刻退回重作。

### 【階段四：品質守門員與進階兵法】

**Day 9: Patric 的火眼金睛 (AI 自我審查與除錯)**
* **實作場景：** Agent 很常產生錯字、幻覺、或不符合「台灣行銷語境 (如把『方案』寫成『項目』)」。
* **實作重點：**
    * 建立 Patric 專屬的審核 Skill 或是 Workflow 的最後一道檢查站 (Checkpoint)。
    * 強制要求所有輸出的文案與 Code，都必須經過 Patric 檢查 SEO 以及無障礙 (WCAG) 規範。
    * **實測：** 故意給一篇充滿大陸用語與錯誤連結的文案，測試 Patric 是否能抓出並修正。

### 【最後驗收】

**Day 10: 期末專案 - 伯大尼「XX公益專案」全自動化發布**
* **實作場景：** 綜合前 9 天所學。給予一項全新的伯大尼行銷挑戰（例如：招募小額定期定額捐款人 500 名）。
* **實戰考核：**
    * 學員需獨立配置完整的 Workflow。
    * 確保工作流涵蓋：市場資料收集 -> 策略擬定 -> 多平台文案生成 -> EDM 個性化 -> 落地頁 (Landing Page) 代碼生成 -> Patric 最終審核。
    * **畢業評鑑：** 按下執行後，檢驗 AI Agent 是否如一個真實運作的「梵亞行銷」分工團隊般，完成整個行銷檔期的籌備工作。

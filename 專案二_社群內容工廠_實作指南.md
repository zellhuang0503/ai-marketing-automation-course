## 專案概述

### 專案目標
建立一個全自動化的 AI 社群內容生產線，從 RSS 新聞監控、選題評分、內容生成、圖片製作到品質檢核，實現 7×24 小時無人值守的內容工廠。

### 核心功能
- **智能選題**: 自動評估 RSS 新聞的社群傳播潛力
- **雙版本內容**: 同時生成正式版與輕鬆版文案
- **AI 圖片生成**: 使用 DALL-E 3 自動製作配圖
- **品質把關**: 三層 AI 檢核確保發布品質
- **Notion 整合**: 完整內容管理與審核流程

## Workflow 1: 內容選題評分系統

### 觸發器設定

#### Trigger 1: RSS Feed (每2小時)
```
節點: RSS Feed Trigger
配置:
  - URL: https://techcrunch.com/feed/
  - Poll Times: 每2小時 (00:00, 02:00, 04:00...)
  - Additional Fields: title, link, content:encoded, pubDate
```

#### Trigger 2: Notion 手動觸發
```
節點: Notion Trigger
配置:
  - Event: Database Item Created/Updated
  - Database: 內容選題資料庫
  - Poll Times: 每5分鐘 (Notion API 限制)
  - 注意: Notion trigger 有 1-5 分鐘延遲，非即時觸發
```

### 節點 1: Agent 1 - 選題評分員

#### 系統提示詞 (System Prompt)
```
你是一位資深的社群媒體選題專家，專精於評估科技新聞的社群傳播潛力。

你的任務是分析輸入的新聞內容，並輸出嚴格的 JSON 格式評估結果。

評分標準 (1-10分):
- 8-10分: 高度相關，強烈建議製作
- 6-7分: 中度相關，可考慮製作
- 1-5分: 相關度低，不建議製作

輸出必須是有效的 JSON，不要包含任何其他文字。
```

#### 用戶提示詞 (User Prompt)
```
請分析以下科技新聞的社群傳播潛力：

標題: {{ $json.title }}
內容: {{ $json.content }}
來源: {{ $json.link }}
發布時間: {{ $json.pubDate }}

請輸出以下 JSON 格式：
{
  "score": 8,
  "reason": "評估理由（50字內）",
  "angle": "建議的傳播角度",
  "target_audience": "目標受眾",
  "platform": "建議發布平台"
}
```

#### AI 節點配置
```
節點: OpenAI Chat Model
模型: gpt-4o-mini
溫度: 0.3 (低溫度確保 JSON 穩定性)
Max Tokens: 500
Response Format: JSON Object
```

### 節點 2: Code 節點 - 解析評分

```javascript
// 節點: Parse Score
// 目的: 從 AI 回應中提取數字評分

const response = $input.first().json.content || $input.first().json.message;

let parsed;
try {
  parsed = JSON.parse(response);
} catch (e) {
  // 如果解析失敗，嘗試提取 JSON 部分
  const match = response.match(/\{[\s\S]*\}/);
  if (match) {
    parsed = JSON.parse(match[0]);
  } else {
    throw new Error('無法解析 AI 回應: ' + response);
  }
}

return [{
  json: {
    score: parsed.score,
    reason: parsed.reason,
    angle: parsed.angle,
    target_audience: parsed.target_audience,
    platform: parsed.platform,
    original_title: $input.first().json.title,
    original_link: $input.first().json.link,
    original_content: $input.first().json.content
  }
}];
```

### 節點 4: Notion - 記錄選題

```

  - Integration Token: [你的 Notion Integration Token]

// 節點: Extract Image URL

const dalleResponse = $input.first().json;

if (dalleResponse.data && dalleResponse.data[0] && dalleResponse.data[0].url) {
  imageUrl = dalleResponse.data[0].url;
} else if (dalleResponse.url) {

return [{

## 常見問題排解

### 問題 1: AI 輸出非 JSON 格式

**症狀**: Code 節點報錯 `JSON.parse()` 失敗

**原因**: 
- AI 模型輸出包含額外文字 (如 "以下是 JSON 格式...")
- 溫度設定過高導致輸出不穩定
- 提示詞未明確要求純 JSON

**解決方案**:
```javascript
// 增強的解析邏輯
const response = $input.first().json.content;

let parsed;
try {
  // 嘗試直接解析
  parsed = JSON.parse(response);
} catch (e1) {
  try {
    // 嘗試提取 JSON 區塊
    const match = response.match(/\{[\s\S]*\}/);
    if (match) {
      parsed = JSON.parse(match[0]);
    } else {
      throw new Error('找不到 JSON 區塊');
    }
  } catch (e2) {
    // 最後嘗試：清理常見問題後解析
    const cleaned = response
      .replace(/```json\s*/g, '')
      .replace(/```\s*/g, '')
      .replace(/^\s*以下是.*?:\s*/gm, '')
      .trim();
    
    try {
      parsed = JSON.parse(cleaned);
    } catch (e3) {
      // 記錄錯誤並返回預設值
      console.error('JSON 解析失敗:', response);
      return [{
        json: {
          error: true,
          raw_response: response,
          score: 5,
          reason: '解析失敗，使用預設值'
        }
      }];
    }
  }
}

return [{ json: parsed }];
```

**預防措施**:
- AI 節點溫度設定為 0.3-0.5
- 明確提示詞: "輸出必須是純 JSON，不要包含任何其他文字"
- 使用 OpenAI 的 `response_format: { "type": "json_object" }`

### 問題 2: DALL-E 3 圖片生成失敗

**症狀**: HTTP Request 節點返回 4xx/5xx 錯誤

**常見原因與解決方案**:

**A. API Key 無效或額度不足**
```
錯誤: 401 Unauthorized
解決: 
1. 檢查 OPENAI_API_KEY 是否正確
2. 登入 OpenAI 平台確認額度: https://platform.openai.com/usage
3. 設定使用上限提醒
```

**B. 提示詞違反內容政策**
```
錯誤: 400 Bad Request - content_policy_violation
解決:
1. 檢查提示詞是否包含敏感內容
2. 避免: 暴力、成人內容、仇恨言論、真實人物
3. 使用更中性的描述詞

範例修正:
❌ "A violent battle scene with blood"
✅ "A strategic chess match between two grandmasters"
```

**C. 請求超時**
```
錯誤: Timeout after 30s
解決:
1. 增加 HTTP Request 節點的超時時間至 120s
2. 啟用重試機制 (Retry): 3次，間隔5秒
3. 考慮使用 n8n 的「Split In Batches」分批處理
```

**D. 圖片 URL 過期**
```
問題: DALL-E 3 返回的 URL 24小時後失效
解決:
1. 立即下載圖片並上傳到永久存儲 (Imgur, AWS S3, Cloudinary)
2. 或使用 n8n 的「Move Binary Data」節點保存到本地

Imgur 上傳範例:
節點: HTTP Request
方法: POST
URL: https://api.imgur.com/3/image
Header: Authorization: Client-ID {{ $env.IMGUR_CLIENT_ID }}
Body (Form Data):
  - image: {{ $json.image_url }} (從 DALL-E 取得)
  - type: URL
```

### 問題 3: Notion 整合問題

**症狀**: Notion 節點報錯或資料未正確寫入

**常見問題**:

**A. Integration 未正確連接資料庫**
```
錯誤: 403 Forbidden - integration not found
解決:
1. 在 Notion 頁面點擊右上角「...」
2. 選擇「Add connections」
3. 搜尋並選擇你的 Integration
4. 確認權限包含「Read content」和「Insert content」
```

**B. 欄位名稱不匹配**
```
錯誤: 400 Bad Request - property not found
解決:
1. 確認 Notion 資料庫欄位名稱與 n8n 設定完全一致
2. 注意: Notion 欄位名稱區分大小寫和全半形
3. 建議: 使用英文欄位名稱避免編碼問題

範例欄位對照:
Notion: "Quality Score" → n8n: "Quality Score"
Notion: "正式版文案" → n8n: "正式版文案"
```

**C. 公式欄位無法寫入**
```
錯誤: 400 Bad Request - formula property is read-only
解決:
1. 公式欄位 (Formula) 由 Notion 自動計算，無法從 API 寫入
2. 如需寫入計算結果，請使用一般文字或數字欄位
3. 或在 Notion 內使用公式引用其他欄位
```

**D. 日期格式錯誤**
```
錯誤: 400 Bad Request - invalid date format
解決:
Notion 日期欄位需要 ISO 8601 格式:
✅ "2026-02-24T10:30:00Z"
✅ "2026-02-24"
❌ "2026/02/24"
❌ "24-02-2026"

n8n 表達式:
{{ $now.format('YYYY-MM-DD') }}
{{ $now.toISO() }}
```

### 問題 4: Webhook 觸發失敗

**症狀**: Workflow 2/3 未被觸發

**排查步驟**:

**A. 確認 Webhook URL 正確**
```
1. 在 Workflow 2 的 Webhook 節點點擊「Test Webhook」
2. 複製顯示的 URL (類似 https://xxx.n8n.cloud/webhook/content-generation)
3. 確認 Workflow 1 中的 HTTP Request 使用相同 URL

```

**B. 檢查 HTTP Request 節點回應**
```
在 Workflow 1 的 HTTP Request 節點後添加「No Operation」節點
檢查輸出是否包含:
- statusCode: 200
- 回應體確認收到

常見錯誤:
- 404: Webhook 路徑錯誤
- 403: n8n 權限設定問題
- 500: Workflow 2 內部錯誤
```

**C. 確認 Workflow 已啟用**
```
在 n8n 介面檢查:
1. Workflow 2 和 Workflow 3 狀態為「Active」(綠色開關)
2. 不是「Test」模式
3. 沒有執行中的錯誤阻擋
```

### 問題 5: RSS Feed 無回應

**症狀**: RSS Trigger 未獲取到新聞

**解決方案**:

## 進階優化建議

### 1. 智能去重機制

```javascript
// 節點: Duplicate Check
// 在 Workflow 1 開頭添加

const newTitle = $input.first().json.title;
const newLink = $input.first().json.link;

// 查詢 Notion 最近 7 天的標題
const recentItems = await $httpRequest({
  method: 'POST',
  url: `https://api.notion.com/v1/databases/${$env.NOTION_DATABASE_ID}/query`,
  headers: {
    'Authorization': `Bearer ${$env.NOTION_API_KEY}`,
    'Notion-Version': '2022-06-28'
  },
  body: {
    filter: {
      timestamp: "created_time",
      created_time: {
        after: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString()
      }
    }
  }
});

// 檢查重複
const isDuplicate = recentItems.json.results.some(item => {
  const title = item.properties.標題?.title[0]?.text?.content || '';
  const link = item.properties.來源連結?.url || '';
  return title === newTitle || link === newLink;
});

if (isDuplicate) {
  return [{ json: { skip: true, reason: '重複內容' } }];
}

return [{ json: { skip: false, ...$input.first().json } }];
```

// 節點: Schedule Post
// 在 Workflow 3 中添加，實現定時發布

const now = new Date();
const hour = now.getHours();

// 根據當前時間決定最佳發布時間
let scheduleTime;
if (hour < 9) {
  // 早上9點發布
  scheduleTime = new Date(now.setHours(9, 0, 0, 0));
} else if (hour < 12) {
  // 中午12點發布
  scheduleTime = new Date(now.setHours(12, 0, 0, 0));
} else if (hour < 18) {
  // 晚上6點發布
  scheduleTime = new Date(now.setHours(18, 0, 0, 0));
} else {
  // 明天早上9點
  scheduleTime = new Date(now.setDate(now.getDate() + 1));
  scheduleTime.setHours(9, 0, 0, 0);
}

return [{
  json: {
    schedule_time: scheduleTime.toISOString(),

    content_casual: $input.first().json.casual,

  }
}];

### 3. 多平台內容適配

```javascript
// 節點: Platform Adapter
// 根據不同平台調整內容格式

const platform = $input.first().json.platform;
const formal = $input.first().json.formal;
const casual = $input.first().json.casual;
const hashtags = $input.first().json.hashtags;

let adaptedContent;

switch(platform) {
  case 'Facebook':
    adaptedContent = {
      text: formal + '\n\n' + hashtags.join(' '),
      max_length: 2000,
      image_required: true
    };
    break;
    
  case 'Instagram':
    adaptedContent = {
      text: casual + '\n\n' + hashtags.join(' ') + ' #ai #科技 #創新',
      max_length: 2200,
      image_required: true,
      aspect_ratio: '1:1'
    };
    break;
    
  case 'Twitter/X':
    adaptedContent = {
      text: casual.substring(0, 200) + '... ' + hashtags.slice(0, 3).join(' '),
      max_length: 280,
      image_required: false
    };
    break;
    
  case 'LinkedIn':
    adaptedContent = {
      text: formal + '\n\n' + hashtags.join(' '),
      max_length: 3000,
      image_required: true,
      tone: 'professional'
    };
    break;
    
  default:
    adaptedContent = {
      text: formal,
      max_length: 2000,
      image_required: true
    };
}

return [{
  json: {
    ...adaptedContent,
    platform: platform,
    original_formal: formal,
    original_casual: casual,
    image_url: $input.first().json.image_url
  }
}];
```

# 專案二：社群內容工廠 - 完整實作指南

## 專案概述

**專案名稱**：社群內容工廠（Multi-Agent Content Factory）  
**難度等級**：⭐⭐⭐⭐（進階）  
**預估時間**：28小時  
**核心技術**：Multi-Agent 協作、n8n Agent Chain、Notion API、AI 圖片生成、RSS 監測

### 功能描述

這是一個**全自動內容生產線**，由 3 個 AI Agent 協作完成：

## 第一部分：環境設定（4小時）

### Step 1：建立 Notion 內容管理資料庫

#### 1.1 建立 Notion 帳號與頁面
1. 前往 https://www.notion.so/ 註冊帳號
2. 建立新頁面，標題：`社群內容工廠`

【截圖位置：Notion 新頁面】

#### 1.2 建立資料庫（Database）
1. 在頁面中輸入 `/database`
2. 選擇「Full page」
3. 標題命名為：`內容生產追蹤`

【截圖位置：建立 Database 選項】

#### 1.3 設定資料庫欄位
點擊「+ New」新增欄位，建立以下欄位：

| 欄位名稱 | 類型 | 選項/說明 |
|---------|------|----------|
| 標題 | Title | 自動 |
| 狀態 | Select | 選項：選題中 / 撰寫中 / 審核中 / 已通過 / 已發布 / 已退回 |
| 原始議題 | Text | RSS 抓到的新聞標題 |
| 選題分數 | Number | Agent 1 評分（1-10） |
| 正式風格文案 | Text | Agent 2 生成 |
| 輕鬆風格文案 | Text | Agent 2 生成 |
| 趣味風格文案 | Text | Agent 2 生成 |
| Hashtag | Text | Agent 2 生成 |
| 圖片提示詞 | Text | Agent 2 生成 |
| 品質分數 | Number | Agent 3 評分（1-10） |
| 審核意見 | Text | Agent 3 的建議 |
| 建立時間 | Created time | 自動 |
| 最終發布平台 | Multi-select | Facebook / Instagram / LINE / 部落格 |

【截圖位置：完整的資料庫欄位設定】

#### 1.4 建立品牌調性頁面
1. 在 Notion 建立新頁面：`品牌調性指南`
2. 內容範例：

```
# 品牌調性指南

## 品牌個性
- 專業但不生硬
- 親切有溫度
- 創意且有趣

## 文案風格
- 多用表情符號 🎉✨💡
- 句子簡短，分段清楚
- 適度使用口語化表達

## 禁止事項
- 不要使用艱深術語
- 避免負面情緒詞彙
- 不要過度推銷

## 目標受眾
- 25-40歲職場工作者
- 對數位行銷有興趣
- 喜歡實用且有趣的內容
```

【截圖位置：品牌調性頁面內容】

#### 1.5 建立 Notion Integration
1. 前往 https://www.notion.so/my-integrations
2. 點擊「New integration」
3. 填寫：
   - Name：`n8n內容工廠`
   - Associated workspace：選擇你的工作區
4. 點擊「Submit」
5. **複製「Internal Integration Token」**（這很重要！）

【截圖位置：Integration Token 顯示位置】

#### 1.6 分享資料庫給 Integration
1. 回到 Notion 的「內容生產追蹤」資料庫
2. 點擊右上角「Share」
3. 點擊「Add connections」
4. 搜尋並選擇剛建立的 `n8n內容工廠`
5. 確認加入

【截圖位置：Share 設定，顯示 Integration 已加入】

#### 1.7 取得 Database ID
1. 開啟「內容生產追蹤」資料庫
2. 看網址列，格式如下：
   ```
   https://www.notion.so/xxxxxxxxx?v=...
   ```
3. `xxxxxxxxx` 這段就是 Database ID（32個字元）
4. 複製並保存

### Step 6：建立 Agent 1 - 選題偵察兵

#### 6.1 添加 RSS Read 節點
1. 在 Trigger 右側點擊「+」
2. 搜尋「RSS」
3. 選擇「RSS Read」
4. 設定：
   - URL：輸入你的 RSS 網址
   - Limit：`5`（每次讀取5篇）

【截圖位置：RSS Read 節點設定】

#### 6.2 添加 AI Agent - 選題判斷
1. 添加「AI Agent」節點
2. Prompt 設定：

```
你是一位專業的內容選題編輯，負責從新聞中找出適合製作社群貼文的議題。

請分析以下新聞，評估其作為社群貼文主題的價值：

標題：{{ $json.title }}
內容摘要：{{ $json.contentSnippet }}

請依照以下格式回覆：

📊 **選題評分**：X/10
（10分=必做，7分以上=值得做，5分以下=跳過）

💡 **選題理由**：
[說明為什麼這個議題適合/不適合]

🎯 **建議切入角度**：
[如果是好選題，建議從什麼角度撰寫]

關鍵字：{{ $json.categories ? $json.categories.join(', ') : '無' }}
```

【截圖位置：Agent 1 Prompt 設定】

#### 6.3 添加 IF 節點 - 篩選高價值選題
1. 添加「IF」節點
2. 設定條件：
   - Mode：`String`
   - Value 1：`{{ $json.content }}`
   - Operation：`Regex`
   - Value 2：`選題評分：([7-9]|10)/10`
   
   （這個正則表達式會抓取 7-10 分的評分）

【截圖位置：IF 節點設定，篩選分數】

#### 6.4 在 Notion 建立選題記錄
對於通過篩選的選題，在 Notion 建立新項目：

1. 添加「Notion」節點 →「Create a Page」
2. 連線：使用 Step 1.5 的 Integration Token
3. 設定：
   - Database：選擇你的 Database ID
   - Title：`= {{ $('RSS Read').item.json.title }}`
   - 狀態：`選題中`
   - 原始議題：`={{ $('RSS Read').item.json.title }}`
   - 選題分數：`={{ $json.content.match(/選題評分：(\\d+)/)?.[1] }}`

【截圖位置：Notion Create Page 設定】

為了讓 Agent 2 在 Notion 有新資料時自動執行，建立第二個 Workflow：

#### 7.2 設定 Notion Trigger
1. 添加「Notion Trigger」
2. 選擇「On New Entry in Database」
3. 設定 Database ID
4. 設定條件：只觸發「狀態 = 選題中」的項目

【截圖位置：Notion Trigger 設定】

#### 7.3 取得品牌調性
1. 添加「Notion」→「Get a Page」
2. 取得品牌調性頁面的內容
3. 或者使用「Set」節點，將品牌調性指南直接寫入變數

【截圖位置：品牌調性資料設定】

#### 7.4 添加 AI Agent - 文案生成
1. 添加「AI Agent」節點
2. 設定 Prompt：

```
你是一位社群文案高手，專門為品牌撰寫吸引人的社群貼文。

【品牌調性指南】
專業但不生硬、親切有溫度、創意且有趣。多用表情符號，句子簡短分段，避免艱深術語。

【選題資訊】
標題：{{ $('Notion Trigger').item.json.properties.標題.title[0].plain_text }}

請生成以下內容：

📝 **正式風格**（適合 LinkedIn、專業場合）
[150-200字，專業口吻]

📝 **輕鬆風格**（適合 Facebook）
[100-150字，親切口吻，多用表情符號]

📝 **趣味風格**（適合 Instagram、年輕族群）
[80-120字，活潑有趣，hashtag 在內文]

#️⃣ **建議 Hashtag**（8-10個）
[混合熱門標籤和長尾標籤]

🎨 **配圖建議**（給 AI 繪圖的提示詞）
[詳細描述適合的圖片風格、場景、色調，英文，50-100字]
```

【截圖位置：Agent 2 Prompt 完整設定】

#### 7.5 生成 AI 圖片
1. 添加「HTTP Request」節點（呼叫 Leonardo API）
2. 或使用「Leonardo AI」節點（如果 n8n 有該節點）
3. 設定：
   - Method：`POST`
   - URL：`https://cloud.leonardo.ai/api/rest/v1/generations`
   - Headers：
     - `Authorization: Bearer {{ $env.LEONARDO_API_KEY }}`
     - `Content-Type: application/json`
   - Body：
     ```json
     {
       "prompt": "{{ $json.content.match(/🎨 \*\*配圖建議\*\*\n([\s\S]+)/)?.[1]?.trim() }}",
       "modelId": "6bef9f1b-29cb-40c7-b9df-32b51c1f67d3",
       "width": 1024,
       "height": 1024
     }
     ```

【截圖位置：Leonardo API 呼叫設定】

#### 7.6 更新 Notion 資料
將生成的內容寫回 Notion：

1. 添加「Notion」→「Update a Page」
2. 設定 Page ID：`{{ $('Notion Trigger').item.json.id }}`
3. 更新欄位：
   - 狀態：`審核中`
   - 正式風格文案：`={{ $json.content.match(/📝 \*\*正式風格\*\*\n([\s\S]+?)(?=\n\n📝 \*\*輕鬆風格)/)?.[1]?.trim() }}`
   - 輕鬆風格文案：`={{ $json.content.match(/📝 \*\*輕鬆風格\*\*\n([\s\S]+?)(?=\n\n📝 \*\*趣味風格)/)?.[1]?.trim() }}`
   - 趣味風格文案：`={{ $json.content.match(/📝 \*\*趣味風格\*\*\n([\s\S]+?)(?=\n\n#️⃣)/)?.[1]?.trim() }}`
   - Hashtag：`={{ $json.content.match(/#️⃣ \*\*建議 Hashtag\*\*\n([\s\S]+?)(?=\n\n🎨)/)?.[1]?.trim() }}`
   - 圖片提示詞：`={{ $json.content.match(/🎨 \*\*配圖建議\*\*\n([\s\S]+)/)?.[1]?.trim() }}`

【截圖位置：Notion Update Page 欄位對應】

**建立 Workflow 3：`專案二_Agent3_品質守門員`**

#### 8.1 設定 Notion Trigger
1. Trigger：`On Entry Update in Database`
2. 條件：狀態變更為「審核中」

【截圖位置：Agent 3 Trigger 設定】

#### 8.2 添加 AI Agent - 品質審核
Prompt 設定：

```
你是一位嚴格的內容品質審核編輯，負責確保文案符合品牌標準。

【品牌調性】
專業但不生硬、親切有溫度、創意且有趣。

【待審核內容】
正式風格：{{ $('Notion Trigger').item.json.properties.正式風格文案.rich_text[0]?.plain_text }}
輕鬆風格：{{ $('Notion Trigger').item.json.properties.輕鬆風格文案.rich_text[0]?.plain_text }}
趣味風格：{{ $('Notion Trigger').item.json.properties.趣味風格文案.rich_text[0]?.plain_text }}

請評估並回覆：

📊 **品質評分**：X/10
（10分=完美，7分以上=可發布，5-6分=小修改，5分以下=大修改）

✅ **符合項目**：
[列出符合品牌調性的地方]

⚠️ **待改進項目**：
[列出需要修改的地方]

💡 **具體建議**：
[給出具體的修改建議]

🔮 **互動率預測**：
高 / 中 / 低（並簡短說明原因）
```

【截圖位置：Agent 3 Prompt 設定】

#### 8.3 IF 節點 - 決定處理方式
1. 添加「IF」節點
2. 判斷品質分數：
   - `Regex` 匹配 `品質評分：([7-9]|10)/10` → **已通過**
   - 其他 → **需修改**

#### 8.4 通過分支：更新狀態並通知
**更新 Notion：**
- 狀態：`已通過`
- 品質分數：`={{ $json.content.match(/品質評分：(\\d+)/)?.[1] }}`
- 審核意見：`={{ $json.content.match(/💡 \*\*具體建議\*\*：\n([\s\S]+)/)?.[1]?.trim() }}`

**發送 Slack 通知：**
1. 添加「Slack」→「Send Message」
2. 選擇「Incoming Webhook」
3. Webhook URL：貼上 Step 2.3 取得的 URL
4. Message：

```
🎉 *新內容已通過審核！*

📌 *標題*：{{ $('Notion Trigger').item.json.properties.標題.title[0].plain_text }}

⭐ *品質評分*：{{ $json.content.match(/品質評分：(\\d+)/)?.[1] }}/10

🔮 *互動率預測*：{{ $json.content.match(/🔮 \*\*互動率預測\*\*：\n(高|中|低)/)?.[1] }}

🔗 *查看詳情*：{{ $('Notion Trigger').item.json.url }}
```

### 常見問題排除

#### 問題 1：RSS 沒有正確讀取
**檢查：**
1. RSS 網址是否正確
2. RSS 格式是否標準（可用 rss.app 測試）
3. 是否有新文章（太舊的 RSS 可能沒有新資料）

#### 問題 2：Notion 更新失敗
**檢查：**
1. Integration 是否有權限（記得 Share）
2. Database ID 是否正確
3. 欄位名稱是否完全一致（大小寫、空格）
4. 資料格式是否正確（rich_text vs title）

#### 問題 3：Agent 評分不準確
**解決：**
- 調整 Prompt，給更具體的評分標準
- 在 Prompt 中加入範例
- 調整 Temperature 參數（降低讓回覆更穩定）

#### 問題 4：AI 圖片生成失敗
**檢查：**
1. Leonardo API Key 有效且有額度
2. Prompt 長度是否符合限制
3. 使用 HTTP Request 節點時，JSON 格式正確

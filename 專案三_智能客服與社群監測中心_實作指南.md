### Step 2：設定 PostgreSQL 資料庫

為了儲存對話記憶，需要資料庫。

#### 2.1 安裝 PostgreSQL（如果還沒裝）

**Windows：**
1. 下載：https://www.postgresql.org/download/windows/
2. 安裝時記得設定密碼
3. 安裝 pgAdmin（圖形化管理工具）

**Mac：**
```bash
brew install postgresql
brew services start postgresql
```

#### 2.2 建立資料庫和資料表
使用 pgAdmin 或命令列：

```sql
-- 建立資料庫
CREATE DATABASE chatbot_memory;

-- 連接到資料庫
\c chatbot_memory

-- 建立對話記憶表
CREATE TABLE conversation_memory (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(100) NOT NULL,
    platform VARCHAR(50) NOT NULL,  -- 'facebook' or 'instagram'
    message TEXT NOT NULL,
    response TEXT,
    intent VARCHAR(50),  -- 'product_inquiry', 'complaint', 'partnership', 'other'
    sentiment VARCHAR(20),  -- 'positive', 'negative', 'neutral'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    context JSONB  -- 儲存額外上下文
);

-- 建立索引加速查詢
CREATE INDEX idx_user_platform ON conversation_memory(user_id, platform);
CREATE INDEX idx_created_at ON conversation_memory(created_at DESC);
```

### Step 3：使用 ngrok 建立公開網址（測試用）

因為 Facebook Webhook 需要公開網址，本地測試時使用 ngrok：

#### 3.1 註冊 ngrok
1. 前往 https://ngrok.com/
2. 註冊帳號
3. 下載 ngrok 程式

#### 3.2 設定 Authtoken
1. 在 ngrok 儀表板取得 Authtoken
2. 在命令列執行：
   ```bash
   ngrok config add-authtoken YOUR_AUTHTOKEN
   ```

#### 3.3 啟動 ngrok
假設 n8n 跑在本機的 5678 port：
```bash
ngrok http 5678
```

會看到類似：
```
Forwarding  https://abc123def.ngrok-free.app -> http://localhost:5678
```

**這個 `https://abc123def.ngrok-free.app` 就是你的公開網址！**

#### 7.2 解析 AI 回覆
添加「Code」節點解析 AI 回覆：

```javascript
const content = $input.first().json.content;

const type_match = content.match(/類型：(\w+)/);
const sentiment_match = content.match(/情緒：(\w+)/);
const human_match = content.match(/人工：(true|false)/);
const reply_match = content.match(/回覆：\n([\s\S]+)/);

return {
  json: {
    intent: type_match ? type_match[1] : 'other',
    sentiment: sentiment_match ? sentiment_match[1] : 'neutral',
    need_human: human_match ? human_match[1] === 'true' : false,
    reply_text: reply_match ? reply_match[1].trim() : '感謝您的訊息，我們會盡快回覆！'
  }
};
```

#### 14.2 查詢統計資料
添加多個 Postgres 節點查詢：

**今日對話統計：**
```sql
SELECT 
    COUNT(*) as total_conversations,
    COUNT(CASE WHEN intent = 'complaint' THEN 1 END) as complaints,
    COUNT(CASE WHEN sentiment = 'negative' THEN 1 END) as negative_count,
    AVG(CASE WHEN sentiment = 'positive' THEN 1 ELSE 0 END) * 100 as positive_rate
FROM conversation_memory
WHERE DATE(created_at) = CURRENT_DATE;
```

**今日輿情統計：**
```sql
SELECT 
    sentiment,
    category,
    COUNT(*) as count,
    AVG(importance_score) as avg_score
FROM sentiment_tracking
WHERE DATE(created_at) = CURRENT_DATE
GROUP BY sentiment, category;
```

### 常見問題排除

#### Webhook 無法接收
1. 檢查 ngrok 是否正常運行
2. 檢查 Facebook Webhook 設定是否正確
3. 檢查 n8n Webhook URL 是否正確

4. 測試用 pgAdmin 能否連線

#### AI 回覆不準確

2. 增加對話歷史的數量（從 5 則增加到 10 則）

### 完整的系統架構

```
使用者（FB/IG）
    ↓
Facebook Webhook
    ↓
ngrok（公開網址）
    ↓
n8n Webhook
    ↓
解析訊息 → 查詢記憶（Postgres）→ AI Agent → 儲存記憶 → 回覆 FB
                                    ↓
                              需要人工？→ 發送警報

排程觸發
    ↓
RSS/API 監測
    ↓
AI 情緒分析
    ↓
儲存分析結果（Postgres）
    ↓
IF 負面且重要 → 即時警報
    ↓
每日彙整 → Google Sheets → Looker Studio
```

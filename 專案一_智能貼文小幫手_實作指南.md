# 專案一：智能貼文小幫手 - 完整實作指南

## 專案概述

**專案名稱**：智能貼文小幫手  
**難度等級**：⭐⭐（基礎）  
**預估時間**：14 小時（實作 7 小時 + 複習 7 小時）  
**核心技術**：n8n Webhook + HTTP Request + OpenAI API + JSON Parsing + Google Sheets  
**文件版本**：v2.0  
**更新日期**：2026 年 2 月

### 功能描述
```
使用者（LINE）
    ↓
輸入：「幫我寫一篇咖啡廳開幕的貼文」
    ↓
Webhook 接收訊息（POST）
    ↓
AI Agent 處理（JSON 輸出）：
├── 判斷適合平台（FB/IG）
├── 生成 3 種風格文案（正式/輕鬆/趣味）
├── 建議最佳發文時間
└── 生成 Hashtag
    ↓
Code 節點解析 JSON
    ↓
自動儲存到 Google Sheets
    ↓
HTTP Request 回覆 LINE（Reply API）
    ↓
發送完成通知到 LINE
```

## 第一部分：環境設定（2 小時）

### Step 1：建立 LINE Bot 帳號

#### 1.1 登入 LINE Developers
1. 前往 https://developers.line.biz/
2. 點擊「Log in to Console」
3. 使用 LINE 帳號登入

#### 1.2 建立 Provider
1. 登入後，點擊「Create New Provider」
2. Provider Name 輸入：`AI行銷課程練習`
3. 點擊「Create」

#### 1.3 建立 Messaging API Channel
1. 點擊剛建立的 Provider
2. 點擊「Create a Messaging API channel」
3. 填寫基本資訊：
   - Channel type：Messaging API
   - Channel name：`智能貼文小幫手`
   - Channel description：`AI自動生成社群貼文`
   - Category：其他
   - Subcategory：其他
4. 勾選同意條款，點擊「Create」

#### 1.4 取得重要憑證

**取得 Channel Access Token：**
1. 左側選單點擊「Messaging API」
2. 找到「Channel access token」區塊
3. 點擊「Issue」產生 Token（選擇長期有效的）
4. **複製這串文字並妥善保存**

**取得 Channel Secret：**
1. 左側選單點擊「Basic settings」
2. 找到「Channel secret」
3. 點擊「Show」顯示
4. **複製這串文字並妥善保存**

**記下你的 Bot ID：**
- 在 Basic settings 頁面，找到「Your user ID」
- 格式類似：`Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

#### 1.5 將 Bot 加入好友
1. 在 Messaging API 頁面，找到「QR code」
2. 用 LINE App 掃描加入好友
3. 發送一則測試訊息給 Bot

### Step 8：設定 Code 節點解析 JSON

#### 8.1 添加 Code 節點
1. 在 AI Agent 節點右側點擊「+」
2. 搜尋「Code」
3. 選擇「Code」節點
4. 點擊「Continue」

#### 8.2 設定 JSON 解析程式碼

選擇 **JavaScript** 模式，輸入以下程式碼：

```javascript
// 從 AI Agent 取得回應內容
const aiResponse = $json.content || $json.text || $json.message?.content || '';

let parsedData = {
  platform: '',
  formal_style: '',
  casual_style: '',
  fun_style: '',
  hashtags: [],
  best_time: ''
};

try {
  // 嘗試解析 JSON
  // 移除可能的 markdown 程式碼區塊標記
  const cleanJson = aiResponse.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();
  parsedData = JSON.parse(cleanJson);
} catch (error) {
  // 如果解析失敗，嘗試用正則表達式提取
  const platformMatch = aiResponse.match(/"platform"\s*:\s*"([^"]+)"/);
  const formalMatch = aiResponse.match(/"formal_style"\s*:\s*"([^"]+)"/);
  const casualMatch = aiResponse.match(/"casual_style"\s*:\s*"([^"]+)"/);
  const funMatch = aiResponse.match(/"fun_style"\s*:\s*"([^"]+)"/);
  const timeMatch = aiResponse.match(/"best_time"\s*:\s*"([^"]+)"/);
  
  parsedData = {
    platform: platformMatch ? platformMatch[1] : '',
    formal_style: formalMatch ? formalMatch[1] : '',
    casual_style: casualMatch ? casualMatch[1] : '',
    fun_style: funMatch ? funMatch[1] : '',
    hashtags: [],
    best_time: timeMatch ? timeMatch[1] : ''
  };
}

// 確保 hashtags 是陣列
if (!Array.isArray(parsedData.hashtags)) {
  parsedData.hashtags = [];
}

// 回傳解析後的資料
return [{
  json: {
    platform: parsedData.platform,
    formal_style: parsedData.formal_style,
    casual_style: parsedData.casual_style,
    fun_style: parsedData.fun_style,
    hashtags: parsedData.hashtags,
    best_time: parsedData.best_time,
    // 原始回應也保留（方便除錯）
    raw_response: aiResponse
  }
}];
```

#### 8.3 設定節點名稱
將節點名稱改為：`解析 AI 回應 JSON`

### Step 10：設定 HTTP Request 回覆 LINE

#### 10.1 添加 HTTP Request 節點
1. 在 Google Sheets 節點右側點擊「+」
2. 搜尋「HTTP Request」
3. 選擇「HTTP Request」節點
4. 點擊「Continue」

#### 10.2 設定 LINE Reply API 請求

**基本設定：**
- **Method**：`POST`
- **URL**：`https://api.line.me/v2/bot/message/reply`
- **Authentication**：`Generic Credential`
  - 點擊「Create New Credential」
  - Name：`LINE_Reply_API`
  - Type：`Header Auth`
  - Header Name：`Authorization`
  - Header Value：`Bearer {{你的Channel Access Token}}`
  - 注意：Bearer 後面要加空格

**Headers（若未使用 Generic Credential）：**
- `Content-Type`：`application/json`
- `Authorization`：`Bearer {{你的Channel Access Token}}`

**Body 設定：**
- **Content Type**：`JSON`
- **Body**：

```json
{
  "replyToken": "{{ $('Webhook').item.json.body.events[0].replyToken }}",
  "messages": [
    {
      "type": "text",
      "text": "✅ 文案生成完成！\n\n📱 建議平台：{{ $('解析 AI 回應 JSON').item.json.platform }}\n\n📝 正式風格：\n{{ $('解析 AI 回應 JSON').item.json.formal_style }}\n\n☕ 輕鬆風格：\n{{ $('解析 AI 回應 JSON').item.json.casual_style }}\n\n🎉 趣味風格：\n{{ $('解析 AI 回應 JSON').item.json.fun_style }}\n\n#️⃣ Hashtags：\n{{ $('解析 AI 回應 JSON').item.json.hashtags.join(' ') }}\n\n⏰ 最佳發文時間：{{ $('解析 AI 回應 JSON').item.json.best_time }}\n\n💾 資料已儲存至 Google Sheets"
    }
  ]
}
```

> ⚠️ **關鍵說明**：
> - 必須使用 `replyToken` 才能回覆，這是 LINE Bot 的限制
> - `replyToken` 有效期只有 1 分鐘
> - 若要主動發訊息（非回覆），需使用 Push API（需要用戶同意）

  ]

### Step 15：創意模式 vs JSON 模式切換

#### 15.1 使用不同 Temperature
| 使用場景 | Temperature | 說明 |
|---------|-------------|------|
| JSON 輸出 | `0.2` | 低溫度確保格式穩定 |
| 創意文案 | `0.7` | 較高溫度產生創意內容 |
| 平衡模式 | `0.5` | 介於兩者之間 |

#### 15.2 若需要創意自由輸出
將 Prompt 改為：
```
你是一位專業的社群行銷文案助手。請發揮創意，
為用戶生成有趣且實用的社群貼文文案。

用戶輸入：{{ $json.body.events[0].message.text }}

請用輕鬆活潑的語氣回覆，不需要拘泥於特定格式。
```
- Temperature 設為：`0.7`
- 不需要 Code 節點解析 JSON
- 直接將 AI 回應傳給 LINE

## 第五部分：常見問題排除

### 問題 1：LINE Bot 沒有回應

**檢查項目：**
1. 確認 n8n Workflow 是「Active」狀態
2. 確認 LINE Webhook URL 正確設定並驗證通過
3. 確認 Webhook 在 LINE Console 是「Enabled」
4. 檢查 n8n 是否有收到請求（Executions 頁面）
5. 若使用 localhost，需使用 ngrok 或 n8n Cloud

### 問題 2：AI 不回應或回應很慢

**檢查項目：**
1. 確認 OpenAI/Gemini API Key 有效且有額度
2. 檢查 n8n 的 log 看有無錯誤訊息
3. 確認 Prompt 中的資料路徑正確
4. 若使用 OpenAI，確認已綁定信用卡並有可用額度

### 問題 3：JSON 解析失敗

**檢查項目：**
1. 在 Code 節點加入除錯輸出，查看原始 AI 回應
2. 確認 Prompt 明確要求 JSON 格式
3. 降低 Temperature 至 0.2
4. 檢查 AI 回應是否有 markdown 程式碼區塊標記（```）

### 問題 4：Google Sheets 沒有寫入資料

**檢查項目：**
1. 確認 Service Account 有正確分享到試算表
2. 確認 Sheet ID 正確
3. 確認欄位名稱和試算表一致
4. 檢查 Code 節點輸出是否有正確的 JSON 欄位
5. 查看 Execution Log 的錯誤訊息

### 問題 5：LINE HTTP Request 回覆失敗

**檢查項目：**
1. 確認使用 `replyToken` 而非 `userId`
2. 確認 Channel Access Token 正確（注意 Bearer 格式）
3. 確認 replyToken 未過期（1 分鐘內有效）
4. 檢查 JSON Body 格式是否正確
5. 查看 HTTP Response 狀態碼：
   - `401`：Token 錯誤或過期
   - `400`：JSON 格式錯誤
   - `429`：請求次數過多

### JSON 解析 Code 節點完整範例

```javascript
const aiResponse = $json.content || $json.text || '';

let result = {
  platform: '',
  formal_style: '',
  casual_style: '',
  fun_style: '',
  hashtags: [],
  best_time: ''
};

try {
  // 清理可能的 markdown 標記
  let cleanJson = aiResponse
    .replace(/```json\s*/gi, '')
    .replace(/```\s*$/gi, '')
    .trim();
  
  result = JSON.parse(cleanJson);
  
  // 確保所有欄位存在
  result.platform = result.platform || '';
  result.formal_style = result.formal_style || '';
  result.casual_style = result.casual_style || '';
  result.fun_style = result.fun_style || '';
  result.hashtags = Array.isArray(result.hashtags) ? result.hashtags : [];
  result.best_time = result.best_time || '';
  
} catch (e) {
  // 解析失敗時，回傳原始內容供除錯
  result = {
    platform: '解析失敗',
    formal_style: aiResponse,
    casual_style: '',
    fun_style: '',
    hashtags: [],
    best_time: '',
    parse_error: e.message
  };
}

return [{ json: result }];
```

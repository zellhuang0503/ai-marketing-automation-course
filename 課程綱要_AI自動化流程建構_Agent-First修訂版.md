# AI自動化流程建構與社群應用實務班 - 課程綱要（Agent-First 修訂版）

> **本文件僅涵蓋黃志宏講師負責的 70 小時術科內容**，為全課程 291 小時術科中的「自動化工作流」核心模組。

## 課程資訊

- **班別名稱**：AI自動化流程建構與社群應用實務班
- **訓練人數**：30人
- **總時數**：385小時（本模組術科 70小時）
- **廠商名稱**：國立台中教育大學
- **修訂日期**：2026年2月
- **修訂重點**：從傳統 Workflow 轉向 Agent-first 思維，採用本地安裝 n8n

---

## 核心教學理念

### 實作導向比例
- **實作操作至少 70%**（理論講授最多 30%）
- 確保30位學員皆能完成 3 個遞進式自動化專案

### 學習目標調整
從「會用 n8n 拉流程」提升至「會設計 AI Agent 系統解決商業問題」

## 三階段專案設計說明

### 專案一：「智能貼文小幫手」（基礎 - 14小時）

含項次 2（7h 專案實作）+ 項次 3（7h 驗收與進階 API）

**核心概念**：AI Agent 基礎 + Webhook 串接  
**學習目標**：學會 Agent 基本運作與 API 工具串接

**技術點：**
- n8n Webhook 節點 + HTTP Request 節點（串接 LINE Bot）
- OpenAI / Gemini API 串接（結構化 JSON 輸出）
- LINE Messaging API（Webhook 接收 + Reply API 回覆）
- Google Sheets 自動記錄

### 專案二：「社群內容工廠」（進階 - 24小時）

含項次 4（14h 專案實作）+ 項次 5（10h 優化與概念導入）

**核心概念**：多 Agent 協作 + 資料流自動化  
**學習目標**：建構完整的內容生產線

**架構（3個 Agent 協作）：**
1. **Agent 1：選題偵察兵** - 監測 RSS、分析關鍵字
2. **Agent 2：文案魔法師** - 生成貼文、Hashtag、配圖提示
3. **Agent 3：品質守門員** - 品牌調性檢查、互動率預測

**技術點：**
- 多 Agent Chain 串接
- Agent Memory（記住品牌調性）
- Notion API 整合
- Slack / Discord Webhook
- OpenAI DALL-E 圖片生成
- API 成本控制

### 專案三：「智能客服與社群監測中心」（高階 - 20小時）

含項次 6（20h 完整專案實作）

**核心概念**：Agent + Workflow 混合架構 + 真實商業場景  
**學習目標**：建構企業級自動化解決方案

**三大模組：**
1. **智能客服 Agent** - LINE/Telegram Bot 自動回覆、問題分類、對話記憶
2. **輿情監測 Agent** - RSS + Google Alerts 關鍵字監測、情緒分析、即時警報
3. **整合儀表板** - Google Sheets + Looker Studio 自動報表、數據視覺化

**技術點：**
- LINE Messaging API / Telegram Bot API（免企業驗證，教學可行性高）
- 進階 Agent 設計（Tool Calling）
- 情緒分析 AI 應用
- Supabase 雲端資料庫（儲存對話歷史，免安裝）
- 進階條件邏輯與錯誤處理

## API 費用規劃

### OpenAI API 費用估算
| 項目 | 模型 | 預估費用 |
|------|------|---------|
| 文案生成 | gpt-4o-mini | ~$0.01/次 |
| 情緒分析 | gpt-4o-mini | ~$0.005/次 |
| 圖片生成 | DALL-E 3 | ~$0.04/張 |

**課程期間預估總費用**：約 $5-10/人

### 建議方案
- **方案 A（推薦）**：課程統一提供一組共用 API Key，設定每人每日使用上限
- **方案 B**：學員自行申請，課程補助每人 $5 額度
- **方案 C**：使用 Google Gemini API 免費方案（每分鐘 15 次，每日 1500 次，足夠課堂使用）

> **重要提醒**：OpenAI 新帳號已無免費試用額度，需預先儲值。Google Gemini API 有免費額度，建議作為備援方案。

## 分組協作機制

### 分組方式
- **6組 x 5人 = 30人**
- 依據電腦狀況與程度異質分組

### 角色分工（每組，每專案輪調）
| 角色 | 人數 | 職責 |
|------|------|------|
| 組長 | 1人 | 協調進度、整合成果、對外發表 |
| 技術 | 2人 | 負責 n8n 流程建置、API 串接 |
| 企劃 | 1人 | 設計內容策略、測試案例、品牌調性 |
| 文件 | 1人 | 整理文件、製作簡報、繪製架構圖 |

### 輪調機制
每個專案結束後輪換角色，確保每位學員都能體驗不同職能。

## 確保30人完成的機制

### 1. 階梯式教學設計
每個專案提供：
- 完整教學影片（課後複習）
- 標準範本檔（可直接匯入 n8n）
- 常見錯誤排除指南
- 進度檢核清單（自我確認）

### 2. 彈性難度設計
| 等級 | 適合對象 | 專案調整 |
|------|---------|---------|
| **基礎版** | 完全新手 | 使用現成模板，少量修改 |
| **標準版** | 一般學員 | 完整實作所有功能 |
| **進階版** | 有經驗者 | 增加額外功能挑戰（如多語言支援） |

### 3. 即時支援機制
**課堂內：**
- 1位講師 + 1位助教巡迴指導
- 分組討論時間（同儕互助）
- 線上共編文件（即時 Q&A）

**課後：**
- LINE 群組即時問答
- 錄影回放（當天上傳）
- 週末線上 Office Hour

### 4. 備援方案
- **共用主機**：3-5 台課堂備援環境
- **兩人一組**：無法安裝者配對協作
- **課後補課**：安排公用電腦練習時間

## 備註

1. 本課綱已針對 2025-2026 AI 發展趨勢調整，聚焦 Agentic AI 而非傳統 Workflow
2. 所有專案均採用本地安裝 n8n，無需平台訂閱費用
3. AI API 費用需額外規劃（建議課程統一提供共用 Key 或補助每人 $5-10）
4. 課程設計確保 30 位學員皆能完成 3 個專案，並有實際成果展示

6. 所有 AI 輸出均採用 JSON 結構化格式，搭配 Code 節點解析，確保輸出穩定
